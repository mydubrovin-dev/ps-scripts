# ps-scripts
Powershell scripts various.

## checkpdf.ps1
Это скрипт который проверяет возможность запуска Acrobat Adobe PDF через DCOM
очень удобен для отладки DCOM
Его задача — попытаться создать COM-объект, открыть PDF-файл (если он указан) и выдать понятный результат.
### Как запустить?
Команда runas со специальным ключом /netonly используется, когда локальный компьютер не введён в домен, но необходимо использовать доменную учётную запись для доступа к сетевым ресурсам. При таком запуске программа локально выполняется от имени текущего пользователя, но для сетевой аутентификации отправляются данные указанной учётной записи.

```cmd
runas /netonly /user:YOURDOMAIN\USR1CV8 "powershell.exe -ExecutionPolicy Bypass -File \"C:\Path\To\checkpdf.ps1\" -PdfPath \"C:\test.pdf\""
```

### Предварительная настройка DCOM и прав

Для успешной работы скрипта от имени USR1CV8 необходимо выполнить следующие действия.

1. Общая настройка прав COM

Это самый важный шаг, так как предоставляет базовые права на запуск и активацию COM-компонентов всем пользователям.

Нажмите Win + R, введите dcomcnfg и нажмите ОК, чтобы открыть Службы компонентов.

В левой панели последовательно раскройте Службы компонентов → Компьютеры → Мой компьютер.

Нажмите правой кнопкой мыши на Мой компьютер и выберите Свойства.

Перейдите на вкладку Безопасность COM.

В разделе Права на запуск и активацию нажмите Изменить значения по умолчанию.

В открывшемся окне нажмите Добавить... и укажите пользователя USR1CV8 (или группу Пользователи, если это допустимо политиками безопасности).

Выберите добавленного пользователя и в нижней таблице разрешений поставьте галочки для Локальный запуск и Локальная активация.

Нажмите ОК для сохранения.

2. Настройка прав для Adobe Acrobat

Некоторые версии Adobe Acrobat могут потребовать явной настройки прав в своей собственной ветке DCOM.

В окне Службы компонентов перейдите в папку DCOM-конфигурация.

Найдите в списке объекты, связанные с Adobe Acrobat. Это могут быть Adobe Acrobat, AcroExch.App или Acrobat Document.

Нажмите на найденный объект правой кнопкой мыши и выберите Свойства.

Перейдите на вкладку Безопасность.

В разделе Права на запуск и активацию выберите Настроить и нажмите Изменить.

Добавьте пользователя USR1CV8 и предоставьте ему права на Локальный запуск и Локальную активацию.

Повторите аналогичные действия в разделе Права доступа, предоставив пользователю USR1CV8 право Локальный доступ.

3. Настройка прав на временные папки

Компоненте AcroExch.PDDoc необходим доступ к системным временным папкам для создания временных файлов. Без этого работа будет невозможна.

Откройте Проводник и перейдите по пути C:\Windows.

Найдите папку Temp, нажмите на ней правой кнопкой мыши и выберите Свойства.

Перейдите на вкладку Безопасность и нажмите Изменить....

Добавьте пользователя USR1CV8, предоставив ему права Чтение и выполнение, Список содержимого папки и Чтение.

Повторите шаги 2-4 для папки C:\Users\Default\AppData\Local\Temp (если она существует).

4. Установка Adobe Acrobat в режиме совместимости

Найдите исполняемый файл Adobe Acrobat (например, Acrobat.exe), нажмите на нём правой кнопкой мыши и выберите Свойства.

Перейдите на вкладку Совместимость.

Поставьте галочку Запускать эту программу от имени администратора.
